1. INTRODUCTION                                              *markpreview-intro*

markpreview.nvim adds an editing/navigation layer for markdown buffers:

  - heading-based folding (ATX `#` and setext `===` / `---`)
  - GFM table editing: align, add/delete rows and columns, edit cells
  - no-wrap by default with comfortable horizontal scrolling, plus a toggle
  - export to PDF / HTML via the bundled `md2pdf` script

It is complementary to in-buffer renderers such as render-markdown.nvim:
that plugin handles how markdown *looks*, this one handles how you *move
through and edit* it.

==============================================================================
2. SETUP                                                     *markpreview-setup*

>lua
    require("markpreview").setup({})
<
With lazy.nvim and a local checkout: >lua
    {
      dir = "/path/to/markpreview.nvim",
      main = "markpreview",
      ft = { "markdown", "markdown.mdx" },
      opts = {},
    }
<
The plugin attaches automatically to the |filetype|s in `config.filetypes`.

3. FOLDING                                                 *markpreview-folding*

Folds nest by heading depth. A `#` heading opens a level-1 fold, `##` a
level-2 fold nested inside it, and so on. Setext headings (a paragraph
underlined by `===` or `---`) are supported too and can be turned off with
`folding.setext = false`; a multi-line setext heading folds from the first
line of its paragraph. Blockquotes, list items and indented code followed by
`===`/`---` are NOT treated as headings (no phantom folds).

Fenced code blocks (``` and ~~~) and YAML/TOML frontmatter are skipped, so
`#` comments in code and `---` separators do not create stray folds.

Folds start open (`folding.default_level = 99`). Use the standard fold
commands: `za` toggle, `zR` open all, `zM` close all, `zj`/`zk` to move
between folds. If folds ever look stale, run |:MarkFoldRefresh|.

4. TABLES                                                   *markpreview-tables*

Place the cursor anywhere inside a pipe table and run any table command or
mapping. The whole table is re-aligned after every edit; column alignment is
read from the delimiter row (`:---` left, `---:` right, `:--:` center).

  |:MarkTableFormat|      align the table
  |:MarkTableCellEdit|    edit the current cell in a floating window
  |:MarkTableRowBelow|    insert an empty row below the cursor
  |:MarkTableRowAbove|    insert an empty row above the cursor
  |:MarkTableColRight|    insert an empty column to the right
  |:MarkTableColLeft|     insert an empty column to the left
  |:MarkTableRowDelete|   delete the current row
  |:MarkTableColDelete|   delete the current column

Rows are always inserted into the body, never above the header or delimiter.
Cell widths use display width, so wide/CJK characters stay aligned. Escaped
pipes (`\|`) inside cells are preserved.

CELL EDITOR                                            *markpreview-cell-edit*

|:MarkTableCellEdit| (`<localleader>tc`) opens a floating window holding just
the cell under the cursor. A cell can't contain a real newline, so each `<br>`
is shown as a separate line — to add a line break, just press Enter. On save
the lines are rejoined with `<br>`, and `|`/`\` are escaped, then the table is
re-aligned. Pipes and backslashes round-trip safely.

  <CR> / q / <C-s>   save and re-align
  <C-c>              cancel
  leaving the window cancels (only the explicit keys above save)

An unchanged cell is left exactly as-is (its `<br/>` spelling, spaces, and any
trailing `<br>` are preserved). The edit is bound to the cell by an |extmark|,
so it still lands correctly if lines shift while the editor is open; if the
table changes underneath, the edit is refused rather than written to the wrong
place.

A line is treated as a table row only when it has a border pipe (it starts or
ends with `|`). This is deliberate: it stops a prose line that merely contains
an inline pipe (e.g. `cmd | grep x`) from being swallowed and reflowed as
cells. Borderless tables are detected once you add the outer pipes / format.

Set `tables.auto_format = true` to re-align the current table automatically
when you leave insert mode. It only fires when the cursor is genuinely inside a
table, so prose with inline pipes is left untouched.

==============================================================================
5. WRAP                                                       *markpreview-wrap*

Markdown windows get `nowrap` plus `sidescroll=1` and `sidescrolloff=8` for
smooth horizontal scrolling of long lines and wide tables. |:MarkWrapToggle|
flips wrap on/off; turning it on also enables `linebreak`+`breakindent` so
prose wraps at word boundaries with hanging indent.

==============================================================================
6. EXPORT                                                   *markpreview-export*

Export the current file to PDF (or self-contained HTML) with no heavy
dependencies. The pipeline is:

    Markdown --(pandoc)--> styled self-contained HTML --(headless browser)--> PDF

|:MarkExportPdf| (`<localleader>p`) and |:MarkExportHtml| run the bundled
`scripts/md2pdf` script asynchronously on the buffer's file (saving it first if
modified) and report the result. When it finishes the result is opened in your
system viewer via |vim.ui.open()|; set `export.open = false` to disable that.
The intermediate HTML uses an embedded print stylesheet (GitHub-like tables,
code highlighting, A4 `@page` margins, row/figure page-break avoidance).

Requirements (all detected at run time):
  - pandoc                          (Markdown -> HTML)
  - a Chromium-family browser       (HTML -> PDF, headless --print-to-pdf)
    google-chrome / chromium / brave / edge — auto-detected

`md2pdf` is usable on its own from any shell: >
    md2pdf notes.md                 # -> notes.pdf
    md2pdf notes.md -o out.pdf
    md2pdf notes.md --html-only      # -> notes.html
    md2pdf notes.md --open           # open the result with xdg-open
<
Put it on your PATH (e.g. symlink `scripts/md2pdf` into `~/.local/bin`).
`wkhtmltopdf` is not auto-selected (its old WebKit paginates poorly); request
it explicitly with `MD2PDF_ENGINE=wkhtmltopdf`.

7. COMMANDS                                               *markpreview-commands*

All commands are buffer-local to attached markdown buffers.

                                                            *:MarkTableFormat*
:MarkTableFormat        Align the table under the cursor.
                                                          *:MarkTableCellEdit*
:MarkTableCellEdit      Edit the current cell in a floating window.
                                                          *:MarkTableRowBelow*
:MarkTableRowBelow      Add an empty body row below the cursor.
                                                          *:MarkTableRowAbove*
:MarkTableRowAbove      Add an empty body row above the cursor.
                                                          *:MarkTableColRight*
:MarkTableColRight      Add an empty column right of the cursor cell.
                                                           *:MarkTableColLeft*
:MarkTableColLeft       Add an empty column left of the cursor cell.
                                                         *:MarkTableRowDelete*
:MarkTableRowDelete     Delete the current row (not header/delimiter).
                                                         *:MarkTableColDelete*
:MarkTableColDelete     Delete the current column.
                                                            *:MarkWrapToggle*
:MarkWrapToggle         Toggle line wrap in the current window.
                                                            *:MarkFoldRefresh*
:MarkFoldRefresh        Recompute heading folds for the buffer.
                                                              *:MarkExportPdf*
:MarkExportPdf          Export this file to PDF (pandoc + headless browser).
                                                             *:MarkExportHtml*
:MarkExportHtml         Export this file to self-contained HTML.

==============================================================================
8. MAPPINGS                                               *markpreview-mappings*

Default buffer-local normal-mode mappings (`<localleader>` is `\` unless you
changed |maplocalleader|):

  <localleader>tf   format/align table
  <localleader>tc   edit current cell (floating window)
  <localleader>tj   add row below
  <localleader>tk   add row above
  <localleader>tl   add column right
  <localleader>th   add column left
  <localleader>td   delete row
  <localleader>tD   delete column
  <localleader>w    toggle wrap
  <localleader>p    export to PDF

Mnemonic: the table keys mirror `hjkl` — j/k add rows below/above, h/l add
columns left/right. Disable all mappings with `keymaps.enable = false`, or
set an individual entry to `false`.

==============================================================================
9. CONFIGURATION                                            *markpreview-config*

Defaults: >lua
    require("markpreview").setup({
      filetypes = { "markdown", "markdown.mdx" },
      folding = {
        enable = true,
        default_level = 99,
        setext = true,
        fold_text = true,
      },
      wrap = {
        enable = true,
        wrap = false,
        linebreak = true,
        breakindent = true,
        sidescrolloff = 8,
      },
      tables = {
        enable = true,
        auto_format = false,
      },
      export = {
        open = true,
      },
      keymaps = {
        enable = true,
        format = "<localleader>tf",
        cell_edit = "<localleader>tc",
        row_below = "<localleader>tj",
        row_above = "<localleader>tk",
        col_right = "<localleader>tl",
        col_left = "<localleader>th",
        row_delete = "<localleader>td",
        col_delete = "<localleader>tD",
        wrap_toggle = "<localleader>w",
        export_pdf = "<localleader>p",
      },
    })
<
